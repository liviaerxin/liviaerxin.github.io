# Diagram Wiki

## Diagram tools

### Graphic-based diagram tools

Features:
- Drag-and-drop functionality.
- A large library of shapes and icons.
- Complex diagram.


Including:

- [Drawio](https://github.com/jgraph/drawio)

- [Lucidchart](https://www.lucidchart.com)

- [Miro](https://miro.com)

### Text-based diagram tools

Features:

- Integration with text editors.
- Version control.
- Less flexible in terms of user interface and graphical capabilities.

Including:

- [Mermaid](https://github.com/mermaid-js/mermaid)
  - Embedded in Markdown or web pages easily.
  - Client-side rendering, based on JavaScript.

- [Plantuml](https://github.com/plantuml/plantuml)
  - Client-side rendering, which needs Java runtime and `plantuml.jar`, and `graphviz` optionally for some diagram.
  - Or Server-side rendering, also needs the above libraries.

- [Asciiflow](https://github.com/lewish/asciiflow)
  - Represent diagrams in pure ASCII TEXT, without any rendering.

### Database schema-based diagram tools

Features:
- Generate diagrams directly from database schema.

Including:
- [ERD Editor | VSCode Extension](https://github.com/dineug/erd-editor)

- [bigER Modeling Tool | VSCode Extension](https://github.com/borkdominik/bigER)

- [MySQL Workbench](https://www.mysql.com/products/workbench/)

- [DbVisualizer](https://www.dbvis.com/)

- [ER/Studio](https://www.idera.com/er-studio-enterprise-architecture-solutions/)

### Creating diagrams on GitHub

You can create diagrams in Markdown using three different syntaxes: `mermaid`, `geoJSON` and `topoJSON`, and `ASCII STL`, said by [GitHub official](https://docs.github.com/en/get-started/writing-on-github/working-with-advanced-formatting/creating-diagrams).
